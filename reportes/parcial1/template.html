Ejercicio Guiado #1

Implementacion de Variables de Entorno (PATH)

Fecha: 12 de Enero, 2026

Autor: Javier Djorkaef Saavedra Mendiola

Tema: Windows y PATH

1. Objetivos

Objetivo General

Comprender la gestion de variables de entorno en windows y modificacion del PATH

Objetivos Específicos

  • Crear un ejecutable ubicado en el Desktop.
  • Añadir el directorio a las variables de entorno
  • Llamar el ejecutable desde la root del disco C:

2. Investigación y Marco Conceptual

Para este ejercicio es necesario entender qué es una Race Condition (Condición de Carrera). Ocurre cuando dos o más procesos acceden a un recurso compartido concurrentemente...

"Un semáforo es una variable protegida que constituye el método clásico para restringir o permitir el acceso a recursos compartidos." — Tanenbaum.

3. Desarrollo Técnico

Para demostrar la utilidad del PATH, creamos un ejecutable simple llamado saludo.exe ubicado en el Escritorio.

// Código C simple compilado como saludo.exe
#include 
int main() {
    printf("¡Ejecución global exitosa desde el Escritorio!\n");
    return 0;
}

Inicialmente, al intentar ejecutarlo desde C:\, el sistema arroja un error. Procedimos a editar las Variables de Entorno de Windows agregando la ruta:

C:\Users\Alumno\Desktop

Una vez aplicada la configuración, el sistema operativo es capaz de resolver la ubicación del ejecutable independientemente del directorio de trabajo actual.

5. Explicación del Código

La función crítica es manejar_recurso(). Aquí utilizamos sem_wait y sem_post. 

void* manejar_recurso(void* arg) {
    // Bloqueamos el semáforo (wait)
    sem_wait(&mutex); 
    
    // Sección Crítica
    contador++; 
    printf("Contador: %d\n", contador);

    // Liberamos el semáforo (signal)
    sem_post(&mutex); 
}
