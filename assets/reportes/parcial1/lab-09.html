Laboratorio en Clase #9

Manejo de Señales en Bash

Fecha: 12 de Febrero, 2026

Autor: Javier Djorkaef Saavedra Mendiola

Tema: Señales e interrupciones en Bash

1. Objetivos

Objetivo General

Comprender el manejo de señales en Bash y la respuesta de un script ante eventos externos.

Objetivos Específicos

  • Implementar funciones que respondan a señales específicas.
  • Utilizar trap para capturar señales como SIGINT y SIGTERM.
  • Mostrar información del sistema al recibir señales.
  • Mantener un script en ejecución esperando señales de manera controlada.

2. Investigación y Marco Conceptual

Señales en Bash

Las señales en Bash permiten que un script reciba notificaciones de eventos del sistema o del usuario. Se pueden usar para interrumpir la ejecución, finalizar procesos o ejecutar funciones específicas.

Trap

La función trap permite asociar funciones del script a señales específicas. Cuando el proceso recibe la señal correspondiente, se ejecuta automáticamente la función definida.

Señales Usadas

  • SIGINT: Señal enviada al presionar Ctrl+C. En este script, muestra la fecha actual.
  • SIGTERM: Señal de terminación. El script muestra los procesos en ejecución con ps.

3. Desarrollo Técnico

Estructura del Script

El script comienza declarando dos funciones: sigint_handler y sigterm_handler. Luego, estas funciones se asocian a las señales SIGINT y SIGTERM mediante trap.

Visualización del PID

Se imprime el PID del script con $$, lo que permite enviar señales externas usando kill -SIGINT <PID> o kill -SIGTERM <PID>.

Bucle de Espera

El script entra en un bucle infinito con while true; do sleep 1; done para mantenerse activo esperando la llegada de señales.

4. Evidencia de Ejecución

Durante la ejecución, al enviar SIGINT con kill -SIGINT <PID>, el script respondió mostrando "SIGINT recibido (Ctrl+C)" y la fecha actual. Al enviar SIGTERM, se mostró "SIGTERM recibido" y la lista de procesos activos con ps.

5. Conclusiones y Trabajo Futuro

El laboratorio permitió entender cómo Bash puede reaccionar ante señales del sistema y del usuario. trap es una herramienta poderosa para manejar interrupciones y acciones personalizadas en scripts.

Trabajo Futuro: Explorar otras señales como SIGALRM, SIGUSR1, SIGUSR2 y crear scripts que realicen tareas automatizadas al recibir dichas señales.
