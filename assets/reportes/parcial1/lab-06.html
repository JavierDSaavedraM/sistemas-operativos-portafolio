Estructuras de control en Bat Scripting

Fecha: 29 de Enero, 2026

Autor: Javier Djorkaef Saavedra Mendiola

Tema: CMD y Windows

1. Objetivos

Objetivo General

Utilizar estructuras de control dentro de Batch Scripting para crear programas simples, conocer las subrutinas y formas de mantener variables locales.

Objetivos Específicos

  • Conocer y utilizar estructuras de control básicas (condicionales y bucles) dentro de los script.
  • Implementar subrutinas y formas de crear funciones internas.
  • Explorar funcionalidades extras modificar variables locales en ciclos.

2. Investigación y Marco Conceptual

Las estructuras de control en Batch Scripting permiten decidir qué instrucciones se ejecutan y en qué orden dentro de un archivo .bat. Gracias a ellas, un script puede tomar decisiones, repetir acciones y responder a distintas condiciones, en lugar de ejecutar comandos de forma lineal.

Entre las más importantes se encuentran IF, que evalúa condiciones; FOR, que permite recorrer archivos, valores o resultados; y GOTO / CALL, que controlan el flujo del programa y el uso de subrutinas. Estas estructuras son la base para crear scripts más organizados, reutilizables y eficientes en el entorno de Windows.

3. Desarrollo Técnico

Paso 1. Estructuras básicas de control.

Para el primer programa se utilizó la estructura IF al igual que un input del usuario para poder escoger una opción.

Todas los datos de entradas que sean obtenidos mediante SET se reciben como string, por lo cual para aceptar valores enteros se debe modificar las opciones.

La estructura del IF sigue un estandar básico en donde tiene un ELSE IF y ELSE dependiendo del resultado.

Paso 2. Estructura de ciclos.

En BATCH existen varios tipos de ciclos que permiten al programa iterar archivos, números, texto o incluso resultados de comandos.

Los diferentes ciclos FOR: FOR IN (VALORES), FOR /F y FOR /L. Cada uno tiene sus diferentes funciones, con la bandera /F el ciclo puede iterar texto dentro de otros archivos y resultados de comandos.

Si la instrucción FOR se utiliza sin ninguna bandera, esta simplemente itera en una lista de valores dados. Por ejemplo (1 2 3 4), itera los valores dados dentro del paréntesis.

Para iterar sobre valores númericos se debe utilizar la bandera /l después de una parentesis con tres valores, (valor inicia, incremento, valor final).

Algo más importante a recalcar es que a diferencia del uso de las variables normales, en los ciclos se utiliza %%f dos signos de porcentaje para representar la variable iteradora.

En el ejemplo06 se representa un loop normal y un loop utilizando valores númericos.

Paso 3. Estructuras GOTO

Dentro de los archivos .bat se pueden crear subrutinas que permiten controlar el flujo del script. Para eso primero se deben crear etiquetas o labels en inglés que permiten identidicar un subconjunto de instrucciones.

Dentro se encuentran dos comandos GOTO y CALL. Ambos cumplen una función similar pero con diferentes propósitos

El comando GOTO permite que el script salte directamente a una subrutina o etiqueta específica, pero no regresa al punto desde el que fue llamado. Por esta razón, dentro de la subrutina se debe redirigir explícitamente el flujo del programa, ya sea hacia el final del script o hacia otra etiqueta, para evitar ejecuciones inesperadas.

En cambio, CALL permite ejecutar una subrutina de instrucciones y, una vez finalizada, regresar automáticamente al punto desde donde fue invocada, continuando con la ejecución normal del script.

Para probar estos conceptos se realizaron 3 programas ejmplos07~ejemplo09. En el primer programa se exploró el manejo básico de CALL, donde siguiendo el flujo del script se hizo un salto a una eqtiqueta para luego regresar y continuar la ejecución normal. Una ves que se termino el programa se hizo un salto al final de archivo para finalizar su ejecución.

5. Conclusiones y Trabajo Futuro

Complementando a las bases de Batch scripting, se puede observar lo útil que es el bash scripting. Aunque sea un lenguaje bastante simple y limitado este ofrece las herramientas suficientes para automatizar lo necesario. Se pueden crear menus, iterar por los archivos y más importante hacer subrutinas de código que se pueden reutilizar.
