Instalacion de WebServer en Fedora

Autor: Javier Djorkaef Saavedra Mendiola

Tema: Linux y web-server

1. Objetivos

Objetivo General

Instalar el servidor Apache en la maquina virtual para crear un web-server

Objetivos Específicos

  • Revisar el comandos para revisar el espacio disponible
  • Instalar el servicio de HTTP (Apache)
  • Inicializar el servicio
  • Hacer que el servicio se inicie automaticamente al iniciar la maquina.
  • Abrir el puerto 80 para que las personas puedan conectarse al servidor

2. Investigación y Marco Conceptual

Apache es un servidor web que permite que una computadora entregue páginas web a los navegadores a través de internet. Cuando alguien escribe una dirección web, Apache recibe la solicitud y responde enviando el contenido del sitio, como textos, imágenes o archivos.

Protocolo HTTPD

El protocolo HTTP (HyperText Transfer Protocol) es el conjunto de reglas que permite la comunicación entre un navegador web y un servidor web. Gracias a HTTP, un usuario puede solicitar una página web y recibirla como respuesta junto con sus recursos, como imágenes, estilos o scripts.

HTTP funciona sobre un sistema de puertos, que son como “puertas de entrada” usadas para identificar el tipo de servicio que se está ofreciendo en un servidor:

  • Puerto 80: Puerto estándar de HTTP (sin cifrado)
  • Puerto 443: Puerto estándar de HTTPS (HTTP seguro con cifrado SSL/TLS)

Servicio en Linux

En Linux, los servicios son programas que se ejecutan en segundo plano (también llamados daemons) y que cumplen una función específica para el sistema o para los usuarios.

Para poder ver los servicios que corren actualmente en el sistema se utiliza el comando sudo systemctl list-units --type=service --state=running

Comandos du y df

Saber administrar los recursos de un servicio es indispensable para los adminsitradores. Existen varias herramientas que permiten el entender el uso actual del disco duro del servidor; como podrian ser los comandos du y df

El comando du (Disk Usage) muestra cuánto espacio ocupan los archivos y carpetas, lo que lo hace ideal para identificar qué directorios están consumiendo más espacio. Es especialmente útil cuando el disco empieza a llenarse y necesitas encontrar el origen del problema.

Por otro lado, df (Disk Free) muestra el estado general del disco, indicando cuánto espacio total hay, cuánto está usado y cuánto queda disponible en cada partición. Este comando es clave para saber si el sistema tiene espacio suficiente para seguir funcionando correctamente.

3. Desarrollo Técnico

Paso 1. Revisar cuando espacio tiene el sistema

Para revisar el estado actual del sistema se puede utilizar el comando df -h como se puede observar en la Figura 1

Paso 2. Instalar el servicio de HTTP (Apache)

Para instalar el servicio se de hacer uso del gestor de paquetes dnf, el cual se encargar de descargar el servidor de manera adecuada.

javier@Fedora:~/

sudo dnf install httpd -y

Paso 4. Inicializar el servicio

Una ves instalado este se debe inicializar con el comando sudo systemctl start httpd.service. Para poder observar el status del servicio se debe utilizar el comando sudo systemctl status httpd.service

Paso 5. Hacer que el servicio se inicie automaticamente al iniciar la maquina

Para hacer que el servicio se inicialice automaticamente cuando la maquina se encienda se debe utilizar el comando sudo systemctl enable httpd.service

Esto hace que se crea un enlace simbolico entre la ubicacion del ejecutable y una carpeta especial que el sistema lee automaticamente al iniciar el servicio.

Paso 6. Abrir el puerto 80 para que las personas puedan conectarse al servidor

Para abir el puerto se debe interactuar con el firewall, el cual se encarga de controlar el trafico de red de la maquina. Para esto se le debe decir que puede aceptar cualquier trafico de red que se intente conectar con el puerto 80.

javier@Fedora:~/

sudo firewall-cmd --add-port=80/tcp --permanent

Esto abre el servicio de tal manera que incluso la maquina host pueda acceder al servicio. En la Figura 3 se puede observar los puertos abiertos de la maquina con firewall-cmd --list-ports y en las Figuras 4 y Figura 5 se puede ver la pagina que muestra que el servidor esta funcionando
