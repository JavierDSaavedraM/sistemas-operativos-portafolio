1. Objetivos

Objetivo General

Familiarizarse con el uso de Batch Scripting en Windows, comprendiendo sus fundamentos básicos y reconociendo los beneficios que ofrece para la automatización de tareas, la optimización de procesos y la mejora de la eficiencia en entornos de trabajo.

Objetivos Específicos

  • Entender el uso de rutas relativas y variables dentro de la cmd.
  • Comprender las instrucciones básicas del Batch Scripting en Windows y su sintaxis general.
  • Analizar el uso de la instrucción @echo off y su importancia en la ejecución limpia de scripts.>
  • Comprender el uso de variables dentro de un script y su ejecucion en un script.
  • Identificar y aplicar mecanismos de input dentro de un script.
  • Conocer y utilizar estructuras de control básicas (condicionales y bucles) dentro de los script.

2. Investigación y Marco Conceptual

Bat Scripting en Windows es la forma más simple de automatizar tareas repetitivas. Con unos cuantos comandos puedes ejecutar procesos, gestionar archivos y ahorrar tiempo directamente desde la línea de comandos. Pequeño en código, grande en productividad. Existe también BASH scripting, el cual se ejecuta en terminales basades en UNIX (Linux y MAC)

Un archivo .BAT es un archivo de programación básica donde se va ejecutando linea por linea el script. No existen a diferencia de programas como C++ o Python, el script no tiene un interprete que revise si la syntaxis es correcta o si existe algunas fallas de lógica. 3. Desarrollo Técnico

Paso 1. Preparación del entorno y bases en comandos.

Para estas practicas se creo una carpeta llamada ejemplos dentro de %USERPROFILE%\Desktop\. Dentro de esta carpeta se crearan los archivos .bat para su ejecución. Para esta primer script se explora los comandos de @echo off y como hacer algunos comentarios.

Dentro de un archivo bat se pueden declarar comentario de dos formas posibles: mediante la palabra clave de REM y al colocar dos puntos al inicio de la linea.::. Con esto en mente se creo el script ejemplos01.bat

Como cada instruccion se ejecuta por linea, este se imprime en la console al tiempo de ejecucion; Sin embargo, se puede desactivar esta opción con @echo off. Además de eso, el comando echo permite al usuario imprimir un string a la pantalla y imprimir una linea vacia.

Paso 2. Variables locales y variables del sistema.

Dentro de un archivo se puede declarar variables con la instruccion set. Para imprimir cualquier variable se debe escribir en conjunto con %%. Estas variables son strings que se pueden utilizar en conjunto con paths para declarar la instalación de aplicaciones o copiar información entre directorio. Por ejemplo se puede utilizar la variable %USERPROFILE% para acceder al root del usuario y de ahí poder acceder a otros directorios. Con esta idea se realiza el programa ejemplo02.bat y ejemplo03.batdonde mediante el path relativo se crea una carpeta y se le asigna una letra para una unidad virutal.

Paso 3. Entrada del usuario

Siguiendo con el comando set se puede utilizar para pedir datos al usuario

Si a una instrucción se le coloca set /p y depués una prompt como string.

De igual manera se puede utilizar el commando pausepara hacer una pausa en el codigo y esperar un enter por parte del usuario.
