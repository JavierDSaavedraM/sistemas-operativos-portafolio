2. Investigación y Marco Conceptual

Dentro de esta practica se usaron muchos concepto, algunos comandos básicos de Linux, conexiones por la red ssh y como funcionan los web-server.

SSH

SSH es un protocolo de transferencia para enviar comandos de forma segura a un dispositivo conectado. El protocolo es una extension del protocolo TPC/IP el cual transfiere paquetes de datos. El puerto default de que utiliza SSH es el 22.

Chmod

Este comando te permite administrar los permisos básicos en Linux. Los permisos estan organizados en los diferentes roles que pueden interactuar con el archivo o directorio (Ejecutar, Leer y Escribir). Estos roles estan dividos en: Users, Group y Others.

El User─o usuario en español─es el dueño del archivo, se representa con la letra u. Normalmente es la persona que creo el archivo por primera vez; aunque, este puede cambiar de ownership con el comando chown. Este rol toma la prioridad más alta al interactuar con el archivo.

Los grupos (Group) son un conjunto de usuarios especificos, este se representa con g. Esto permite que varios usuarios puedan pertenecer al mismo grupo y compartir los archivos sin hacerlos publicos. Comandos como groups o groupadd permiten interactuar con Para que un grupo pueda tener acceso en un directorio el super usuario tiene que hacer que un directorio sea propiedad del grupo.

Finalmente others hace referencia a todos los usuarios del sistema que no sean el dueño del archivo o no esten en grupo especifico. Este se representa con la letra o.

Los permisos son mostrados en una cadena de 10 caracteres cuando se ejecuta el comando ls -l. En la Figura 1 se muestran los posibles valores que puede obtener la cadena.

Este se divide en 4 secciones. El inicio de la cadena le dice al usuario si el archivo es un directorio o un archivo. Después se divide en caracteres de 3. El primer conjunto representa los permisos del usuario, el segundo los permisos del grupo y el ultimo los restantes.

Los permisos son representados mediante las letras r, w, x; para las operaciones leer, escribir y ejecutar respectivamente. Para poder cambiar se utiliza el siguiente comando.

624198_users@fedora:~/

$ chmod g+w html

El primer parametro es el rol al que va el permiso. Siguiente es un + o un - representando otorgar o revocar y finalmente al archivo o carpeta. Para poder darle más permisos a otros roles.

De igual manera se puede representar los permisos de manera númerica.Como se puede observar en la Figura. Para poder combinarlos se suman; Por ejemplo al escribir > chmod 755 file.txt; Hace que el el rol de owner pueda ejectura, leer y escribir, mientras que los grupos y los demas puedan solamente verlo y ejecturalo.

Chcon

Este comando te permite cambiar el contexto de la segurida de un archivo. Dentro de un ambiente SELinux (Ambiente de seguridad), cada archivo tiene una label que describe los archivos como objetos. Cada objeto tien une un user, rol, typo y nivel. Por lo que para poder cambiar.
