Laboratorio en Clase #1

Implementación de Variables de Entorno (PATH) en Windows

Fecha: 12 de Enero, 2026

Autor: Javier Djorkaef Saavedra Mendiola

Tema: Windows y PATH

1. Objetivos

Objetivo General

Comprender la gestión de variables de entorno en Windows y modificación del PATH. Se busca configurar el sistema para que ejecute un programa situado en el escritorio desde cualquier ubicación en el sistema.

Objetivos Específicos

  • Crear un ejecutable ubicado en el Desktop.
  • Añadir el directorio a las variables de entorno.
  • Llamar el ejecutable desde la raíz del disco C:.

2. Investigación y Marco Conceptual

Para este ejercicio es necesario entender el concepto de una variable de entorno. Según Wikipedia una variable de ambiente es un valor dinámico que ayuda a los programas y procesos a tener la información sobre el entorno de su ejecución.

Las variables se pueden dividir en variables locales y globales. Al ser una variable global, esta afecta al sistema entero. Mientras que una variable local solamente afecta el ambiente del usuario trabajando.

Un ejemplo bastante claro es la variable PATH. Esta guarda una lista de carpetas y le dice al sistema donde buscar programas ejecutables.

3. Desarrollo Técnico

Para demostrar la utilidad del PATH, creamos un ejecutable simple llamado sisop.exe ubicado en el Escritorio.

// Código C++ simple compilado como sisop.exe
#include <iostream>
int main() {
   std::cout << "Esto es mi programa" << std::endl;
    return 0;
}

Inicialmente, al intentar ejecutarlo desde C:\, el sistema arroja un error. Procedimos a editar las Variables de Entorno de Windows agregando la ruta:

C:\Users\djjav\Desktop

Una vez aplicada la configuración, el sistema operativo es capaz de resolver la ubicación del ejecutable independientemente del directorio de trabajo actual.
