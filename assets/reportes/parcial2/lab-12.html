2. Investigación y Marco Conceptual

Procesos

Un proceso es una instancia en ejecución de un programa. Cada proceso tiene su propio espacio de memoria, recursos asignados y un identificador único llamado PID (Process ID).

A diferencia de un programa, que es solo un archivo almacenado en disco, un proceso representa al programa en ejecución dentro del sistema operativo. Un proceso puede tener varios estados:

  • running: el proceso se está ejecutando en la cpu.
  • ready: el proceso está listo para ejecutarse.
  • waiting: el proceso espera un evento (e/s, señal).
  • stopped: el proceso fue detenido manualmente.
  • paused: el proceso se encuentra temporalmente suspendido.

Procesos padre hijo y Zombies

Dentro de Linux exite lo que son los proceso padres e hijos. Un proceso padre es el proceso original que crea otro proceso, y un proceso hijo es la copia que nace a partir de él.

Esto ocurre mediante la llamada fork(). Cuando se ejecuta, el sistema clona el proceso. Ambos continúan desde el mismo punto del programa, pero fork() devuelve un valores distintos para diferenciarlos: el padre recibe el PID del hijo y el hijo recibe 0.

Un proceso hijo se queda en la lista de proceso incluso cuando ya termino su ejecucion, esto pasa porque linux le permite que los padres tengan un estatus de sus hijos. Para eso existe la funcion de waitpid() esta permite que el padre pueda estar al pendiente que su hijo termino y que lo puede eliminar.

Si un el padre no termina mata a el hijo cuando este completa su funcion. El hijo se vuelve defunct o en otras palabras en un estado zombie. Este es un proceso que no toma recursos pero se encuentra en la lista de procesos.

En linux se puede utilizar el comando pstree para ver todos los proces en forma de arbol. Esto permite ver los procesos padre e hijo

En Linux, cada proceso tiene su propio espacio de direcciones virtuales dividido en varios segmentos de memoria. Cada segmento guarda un tipo diferente de datos o instrucciones del programa.

  • Text Segment: Contiene el código en lenguaje máquina.
  • Data Segment: Variables globales inicializadas.
  • BSS: Variables globales no inicializadas.
  • Heap: Memoria dinámica reservada en tiempo de ejecución, se puede modificar con funciones malloc() calloc() y free(), dentro de c y c++.
  • Stack: Pila de ejecución (funciones y variables locales).

Par ver la estructura de la memoria de los procesos se tiene que ver el contenido del directorio: /proc/[PID]/maps. Dentro de /proc se encuentran todos los procesos corriendo en el sistema. Los directorios estan divididos por su PID. Dentro de cada uno se encuentra ./maps es lo cual tiene toda la informacion del HEAP y el STACK.

Links Simbolicos

Un link simbolico es un archivo de linux que apunto a otro archivo o directorio dentro de la computadora. Existen como un "shortcut" en Windows. Existen dos tipos de links, los hardlinks y los softlinks. La principal diferencia entre ellos es que los hardlinks no permiten que un acceso a otra carpeta que no este en el sistema.

Para crear un link simbolico se debe utilizar el comando ln despues una bandera que indique si es hard link o softlink y al final el path del archivo o carpeta. ejemplo de softlink: ln -s [original] [nuevoLink]

Dentro de linux hay una forma de hacer que los hijos de un proceso padre se conviertan en zombie. Cuando dentro de un proceso que esta esperando hijos mato al padre este deja a los hijos huerfanos.

Paso 2. Ver el mapeo de memoria

Para poder ver el mapeo de la memoria en el sistema operativo se puede necesita ver el contenido de /proc/[PID]/maps. Esto se puede ver mediante el comando de `grep -E '\[heap\]|\[stack\]' /proc/4439/maps` , aunque este mostrara un formato que los humanos no pueden leer (Figura ). Este comando muestra el contendio dentro del STACK y el HEAP.

El comando cat /proc/[PID]/task/[PID]/maps | grep '\[stack]' en cambio solamente muestra el STACK de proceso.

Paso 3. Ver las regiones de memoria (HEAP y STACK) pmap y smap

De igual manera se tiene el comando pmap -x [PID] | grep -E '\[heap] | \[ stack ]' que consigue lo mismo pero con con formato que se puede entender. Te muestra las regiones de la memoria, el tamaño de cada region, la memoria del RAM y los archivos que estan asociados.
